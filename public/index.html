API REST Sample made with NestJS / ExpressJS

Características

  • Puerto establecido desde environments
  • Cors abierto
  • Statics disponible desde la carpeta public
  • ORM Prisma para acceder a una DB PostgreSql

  • CRUD de tareas

    • Entities definidas como interfaces
    • Dtos definidos como clases
    • Validación basada en el pipe ValidationPipe
    • Librerías de validación class-validator y class-transformer

Endpoints

  • [GET] /tasks
  • [GET] /tasks/:id
  • [POST] /tasks
  • [PATCH] /tasks/:id
  • [DELETE] /tasks/:id
  • [GET] /users
  • [GET] /users/:id
